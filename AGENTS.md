# Agent Guide

This is the Quark JS SDK — a unified TypeScript client for the Quark
platform. It speaks gRPC to the four Quark components (auth, server,
node, workflow) over the Connect-RPC wire protocol using
`@connectrpc/connect` + `@connectrpc/connect-web`. Every RPC is fully
typed; types are generated from .proto contracts via `buf generate`.

## Repository

- **Name**: Quark JS SDK
- **Language**: TypeScript (ESM-only, no build step)
- **License**: MIT
- **Wire protocol**: Connect-RPC (JSON or binary encoding over HTTP/1.1+)
  via `@connectrpc/connect` and `@connectrpc/connect-web`.
- **Type generation**: `@bufbuild/protoc-gen-es` (v2 codegen) — generates
  message types and service descriptors from `proto/*.proto` into
  `src/gen/*_pb.ts`.

## Quick reference

```bash
# Install dependencies (note: --legacy-peer-deps is NOT required after
# the v2 codegen migration — we no longer depend on @connectrpc/protoc-gen-connect-es)
npm install

# Regenerate src/gen/*_pb.ts from proto/*.proto
npm run generate

# Regenerate the typed service wrappers in src/services/*.ts
python3 scripts/generate_quark_js_services.py

# Type-check (must pass with zero errors)
npm run typecheck
```

```bash
# Verify the public API surface (what's exported from index.ts)
node --input-type=module -e "import * as api from './src/index.ts'; console.log(Object.keys(api).sort().join('\n'))"

# Sanity-check that no internal modules leaked into the public API
grep -E "export.*createQuarkTransportInternal|export.*connectJsonUnary" src/index.ts
# (should return nothing — only the documented public API is exported)
```

## Proto lifecycle

The .proto files in `proto/` are duplicate copies of the source-of-truth
protos owned by the upstream service repositories:

| File | Source |
|---|---|
| `proto/auth.proto`, `proto/common/*.proto` | github.com/quarkloop/auth |
| `proto/server.proto`, `proto/nodes.proto`, `proto/release.proto`, `proto/secrets.proto`, `proto/workflow.proto` | github.com/quarkloop/server |
| `proto/node.proto` | github.com/quarkloop/quark-rs (crates/quark-node-proto) |
| `proto/google/protobuf/*.proto` | protoc runtime include (vendored for offline codegen) |

When a proto changes upstream:

1. **Copy the new proto** into `proto/` (must match upstream byte-for-byte).
2. **Run `npm run generate`** — invokes `buf generate` which runs
   `@bufbuild/protoc-gen-es` to regenerate `src/gen/*_pb.ts`.
3. **Run `python3 scripts/generate_quark_js_services.py`** — regenerates
   the typed service wrapper classes in `src/services/*.ts` by parsing
   the generated descriptors.
4. **Run `npm run typecheck`** — verify the SDK still compiles.

Do NOT edit `src/gen/*_pb.ts` or `src/services/*.ts` by hand. The first
are auto-generated by `buf generate`; the second are auto-generated by
the python script. To change a service wrapper, edit the python script
and re-run it.

One responsibility per file. The dependency graph is strictly acyclic:

```
index.ts
  ↓
client-builder.ts ──► client.ts ──► services/auth.ts  ─┐
                  └──► services/server.ts  ────────────┤
                  └──► services/node.ts    ────────────┤
                  └──► services/workflow.ts ───────────┤
                                                       ↓
                                              transport.ts
                                                       ↓
                                                   errors.ts

services/*.ts ──► gen/*_pb.ts (generated, read-only)
```

`ServiceClient` (the abstract base for every gRPC service wrapper) lives in
`transport.ts` so that `auth.ts`, `server.ts`, `node.ts`, and `workflow.ts`
share it without cross-importing each other.

## Codegen version compatibility — READ THIS

The v2 codegen line (`@bufbuild/protobuf@2.x` + `@bufbuild/protoc-gen-es@2.x`
+ `@connectrpc/connect@2.x`) is a clean break from v1. **Do NOT mix v1 and v2
packages** — it produces service descriptors that fail TypeScript's
`DescService` constraint with cryptic "missing properties: kind, name, file,
method" errors.

### What's compatible

| Package | Required version line |
|---|---|
| `@bufbuild/protobuf` | `^2.7.0` (runtime) |
| `@bufbuild/protoc-gen-es` | `^2.7.0` (codegen, devDep) |
| `@connectrpc/connect` | `^2.0.0` (runtime) |
| `@connectrpc/connect-web` | `^2.0.0` (runtime) |

### What's NOT compatible

- `@connectrpc/protoc-gen-connect-es@1.x` — generates v1-style `*_connect.ts`
  descriptors that DON'T satisfy the v2 `DescService` interface. Do NOT add
  this package. The v2 `protoc-gen-es` plugin generates service descriptors
  directly in `*_pb.ts`, making the separate `*_connect.ts` file obsolete.

### Codegen output layout

`buf generate` produces ONE file per .proto:

- `proto/auth.proto` → `src/gen/auth_pb.ts` (messages + service descriptors)
- `proto/server.proto` → `src/gen/server_pb.ts` (messages + service descriptors)
- `proto/node.proto` → `src/gen/node_pb.ts` (messages + service descriptors)
- `proto/workflow.proto` → `src/gen/workflow_pb.ts` (messages + service descriptors)

Each `*_pb.ts` exports:

- One `GenMessage<T>` per proto message (e.g. `LoginRequest`, `LoginResponse`)
- One `GenService<T>` per proto service (e.g. `AuthService`, `UserService`)

The service descriptors can be passed directly to
`createClient(AuthService, transport)` from `@connectrpc/connect`.

## Service wrapper pattern

Every service class in `src/services/*.ts` follows the same pattern:

```ts
import { createClient, type Client } from '@connectrpc/connect';
import { AuthService as AuthServiceDesc } from '../gen/auth_pb.js';
import type { LoginRequest, LoginResponse } from '../gen/auth_pb.js';
import type { QuarkTransport, QuarkCallOptions } from '../transport.ts';
import { ServiceClient } from '../transport.ts';

export class AuthService extends ServiceClient {
  private readonly client: Client<typeof AuthServiceDesc>;

  constructor(transport: QuarkTransport) {
    super(transport);
    this.client = createClient(AuthServiceDesc, transport.underlying);
  }

  login(request: LoginRequest, options?: QuarkCallOptions): Promise<LoginResponse> {
    return this.client.login(request, options);
  }
  // ... one method per RPC
}
```

- The generated descriptor is imported with an `as <Name>Desc` alias to avoid
  clashing with the SDK wrapper class name (which is the same).
- The wrapper class holds a typed `Client<typeof Descriptor>` created via
  `createClient()`.
- Each method delegates to `this.client.<rpcName>(request, options)` — fully
  typed, no `unknown` anywhere.
- For services whose input is `google.protobuf.Empty`, the method takes no
  request argument: `health(options?: QuarkCallOptions): Promise<HealthStatus>`.

The `AuthClient` and `ServerClient` facades extend their respective base
service classes (e.g. `AuthClient extends AuthService`) so `quark.auth().login()`
works without a double-call. Other services on the same component are accessed
via lazy accessors (`quark.auth().users()`, `quark.server().organization()`, …).

## Public API

The public API is exported from `src/index.ts`. It includes:

- `QuarkClient`, `QuarkClientBuilder`, `QuarkClientConfig`
- `QuarkError` + 8 subclasses + `fromConnectError` + helpers
- Service wrapper classes: `AuthClient`, `AuthService`, `UserService`, …
- `ServerClient`, `ServerService`, `OrganizationService`, …
- `NodeService`, `WorkflowService`
- Transport-layer types: `QuarkTransport`, `QuarkTransportOptions`,
  `QuarkProtocol`, `QuarkCallOptions`, `QuarkHeadersInit`,
  `createQuarkTransport`, `ServiceClient`
- Generated type namespaces: `authGen`, `serverGen`, `nodeGen`,
  `workflowGen`, `commonGen` — re-export the contents of
  `src/gen/*_pb.ts` under named barrels to avoid name collisions between
  proto packages.

## Common mistakes

- **Adding `@connectrpc/protoc-gen-connect-es` back.** It generates v1-style
  `*_connect.ts` files that are incompatible with `@bufbuild/protobuf@2.x`.
  The v2 `protoc-gen-es` plugin already generates service descriptors in
  `*_pb.ts` — no separate connect-es plugin is needed.

- **Using `headers` option in `createConnectTransport`/`createGrpcWebTransport`.**
  These transports do NOT accept a `headers` field. Default headers must be
  injected via an interceptor. See `defaultHeadersInterceptor` in
  `src/transport.ts` for the canonical pattern.

- **Using `HeadersInit` as a type.** Under `lib: ["ES2022"]` (no DOM),
  `HeadersInit` is not a top-level global even though `Headers` is. Use the
  local `QuarkHeadersInit` type from `transport.ts` instead.

- **Editing `src/gen/*_pb.ts` or `src/services/*.ts` by hand.** Both are
  auto-generated. To change a service wrapper, edit
  `scripts/generate_quark_js_services.py` and re-run it. To regenerate
  types from protos, run `npm run generate`.

- **Forgetting to sync `proto/` with upstream.** The protos in `proto/`
  must be byte-for-byte identical to the source-of-truth protos in the
  upstream service repos. When upstream changes, copy the new proto into
  `proto/` and re-run codegen.

- **Forgetting to bump the codegen script when a new RPC is added.** The
  python script parses `src/gen/*_pb.ts` to enumerate RPCs. If a new RPC
  is added to a proto and the script isn't re-run, the SDK will be missing
  the new method. The script is idempotent — re-run it any time the protos
  change.

## Error model

Every failure path is normalised to a `QuarkError` subclass via the
`errorNormalisingInterceptor` in `transport.ts`. This interceptor wraps
every outgoing RPC and converts `ConnectError` (and other shapes) to the
appropriate `QuarkError` subclass via `fromConnectError`.

`fromConnectError` maps Connect `Code` values to `QuarkError` subclasses:

| Connect Code | QuarkError subclass |
|---|---|
| `Code.Unauthenticated` (16) | `UnauthenticatedError` |
| `Code.NotFound` (5) | `NotFoundError` |
| `Code.PermissionDenied` (7) | `PermissionDeniedError` |
| `Code.AlreadyExists` (6) | `AlreadyExistsError` |
| `Code.Unavailable` (14) | `UnavailableError` |
| `Code.DeadlineExceeded` (4) | `DeadlineExceededError` |
| `Code.InvalidArgument` (3) | `InvalidArgumentError` |
| Network failure / `TypeError` / `DOMException` abort | `ConnectionError` or `DeadlineExceededError` |
| Anything else | `QuarkError` (`code: 'UNKNOWN'`) |
